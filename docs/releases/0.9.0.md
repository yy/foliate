# Foliate 0.9.0

Foliate 0.9.0 makes watch startup incremental, improves Quarto rendering, and moves optional generated-asset publication into deployment.

## Highlights

- Watch startup reuses valid HTML, Quarto renders, source visibility metadata, and postprocessing results. It still invalidates the relevant cache when a source, template, configuration file, or public-page set changes.
- Quarto execution results persist across prose edits. `foliate build --force` refreshes executable cells, while documents with inline expressions render without Jupyter Cache because Quarto does not support that combination.
- Renaming or deleting a QMD source now removes its stale rendered Markdown and Foliate-generated Obsidian preview. Hand-written files in the preview directory are preserved.
- Generated figures keep stable modification times when their contents do not change. Local sites continue to store figures in `assets/quarto/`; sites with a publisher configuration store them in `.foliate/cache/quarto/assets/` and publish them during deployment.

## Migrating the generated-asset publisher

Version 0.8 published one page at a time through `foliate publish-assets`. Version 0.9 removes that command and publishes the complete referenced asset tree during `foliate deploy`.

Remove `publish_assets` from page frontmatter and replace the v0.8 publisher keys:

```toml
[publisher]
command = ["aws", "s3", "cp", "{source}", "s3://bucket/{key}"]
url_template = "https://cdn.example/{key}"
key_template = "{page_slug}/{filename}"
draft_root = "assets/drafts/quarto"
manifest = ".foliate/published-assets.json"
```

with the v0.9 tree-publishing configuration:

```toml
[publisher]
public_base_url = "https://cdn.example/site-assets"
key_prefix = "quarto"
command = [
  "aws", "s3", "sync",
  "{staging_prefix_dir}", "s3://bucket/site-assets/{key_prefix}",
  "--delete",
  "--cache-control", "no-cache",
]
```

The command must contain `{staging_dir}` or `{staging_prefix_dir}`. Foliate stages only generated assets referenced by public pages, rewrites their URLs in a deployment copy, then invokes the command. Normal builds and watch previews do not upload files.

Run `foliate deploy --dry-run` after migrating. It prepares the deployment copy and reports the generated assets without invoking the publisher.

## Deploy now builds by default

`foliate deploy` now builds the site before syncing the deployment target. Use `foliate deploy --no-build` when you intentionally want to deploy the existing `.foliate/build/` directory.
